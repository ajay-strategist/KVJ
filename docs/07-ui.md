# 07 — UI Documentation

## Design system

- **Styling:** Tailwind CSS v4 with a small `@theme` token set in `index.css` — `--color-primary #1A3E6F` (deep navy), `--color-accent #2E75B6` (blue), `--color-base #FFFFFF`, `--color-soft-grey #F8FAFC`, and the Inter font. No component library (no MUI/AntD/shadcn); UI is hand-built with Tailwind utility classes.
- **Icons:** lucide-react.
- **Responsiveness:** Tailwind responsive utilities are used, but there is no evidence of a systematic mobile-first pass; the dense admin tables and the chat/kanban layouts are desktop-oriented.
- **Dark mode:** **Not supported.** No `dark:` variants, no theme toggle, no `prefers-color-scheme` handling found. The palette is a single light theme.
- **Forms & validation:** forms are plain controlled React inputs; validation is **ad-hoc and inline** (manual checks before submit) — no form library (Formik/RHF) and no schema validation (Yup/Zod). Server-side, Mongoose enums/required act as the backstop.
- **State:** local `useState`/`useEffect` per page; the only shared context is `SocketContext`. No global store, no data-fetching cache (React Query/SWR).
- **Routing/navigation:** React Router v7. Navigation is via the sidebar in `DashboardLayout` (Link + `useNavigate`); the sidebar renders role-appropriate items.

## Public / marketing screens

### Home (`/`)
- **Purpose:** marketing landing page.
- **Components:** Navbar, Hero, Features, HowItWorks, SocialProof, ChatHighlight, TeamStatusPanel, Testimonials, Pricing, CtaBanner, Footer.
- **Actions:** navigate to Login/Signup.
- **Dependencies:** `useFadeIn` hook for scroll animations. No API/auth.

### Login (`/login`)
- **Purpose:** authenticate.
- **Components:** email/password form.
- **Actions:** POST `/api/auth/login`; on success stores `token`+`user` in `localStorage` and redirects to a dashboard.
- **Dependencies:** `API_BASE_URL`, axios, `localStorage`.

### Signup (`/signup`)
- **Purpose:** self-register (creates Pending account).
- **Components:** registration form.
- **Actions:** POST `/api/auth/register`; shows "awaiting admin approval".
- **Dependencies:** axios.

## Authenticated application screens

All wrap themselves in `DashboardLayout` (sidebar + toasts + heartbeat).

### Dashboard (`/dashboard`)
- **Purpose:** employee/manager home — personal snapshot.
- **Components:** stat cards, quick links; presence widgets.
- **Actions:** navigate to modules; may show my tasks/attendance/leave summary.
- **Dependencies:** attendance/task/leave APIs, SocketContext.

### AdminDashboard (`/dashboard/admin`)
- **Purpose:** admin KPI overview.
- **Components:** stat cards (active users, projects, today's attendance, pending expense total), pending leaves & expenses queues.
- **Actions:** drill into approvals.
- **Dependencies:** `GET /api/dashboard/admin`.

### AdminUsers (`/dashboard/users`)
- **Purpose:** user administration (866 LOC).
- **Components:** user table, approve/reject controls, role/team/grade/salary editors, trainer toggle, export button, performance-report launcher.
- **Actions:** approve, reject, update, deactivate, toggle trainer, export CSV/XLSX (export opens `?token=` URL in a new tab), open performance report.
- **Dependencies:** `/api/users/*`, `/api/admin/users/:id/trainer`, teams API, SocketContext (`userUpdated`, `forceLogout`).

### ManageBatches (`/dashboard/batches`)
- **Purpose:** manage colleges, courses and training batches.
- **Components:** lists + create forms for College/Course/Batch.
- **Actions:** create/update/delete batches, add colleges/courses.
- **Dependencies:** `/api/training-batches/*`.

### Teams (`/dashboard/teams`)
- **Purpose:** team CRUD & membership.
- **Components:** team cards/table, manager selector, member view.
- **Actions:** create/update/delete team (Admin), view members.
- **Dependencies:** `/api/teams/*`, users API.

### Projects (`/dashboard/projects`)
- **Purpose:** projects + task board (**largest screen, 1,488 LOC**).
- **Components:** project list, task board/columns by status, task cards, task create/edit modals, comments, timer controls, transfer controls, filters.
- **Actions:** create/update/delete projects & tasks, claim pool tasks, log time, start/pause/end timer, comment, initiate/accept/reject transfers, archive.
- **Dependencies:** `/api/projects/*`, `/api/tasks/*`, `/api/tasks/:id/timer/*`, settings (labels), SocketContext (`poolTaskClaimed`, task/project updates).

### ProjectManagement (`/dashboard/project-management`)
- **Purpose:** portfolio-level project management view.
- **Components:** project overview table/cards, status, members/managers.
- **Actions:** manage projects and assignments.
- **Dependencies:** `/api/projects/*`.

### Attendance (`/dashboard/attendance`)
- **Purpose:** personal clock-in/out (thin, 24 LOC — likely delegates to shared logic/components).
- **Components:** clock in/out, break in/out, geolocation prompt, today's summary.
- **Actions:** clock-in/out, break-in/out.
- **Dependencies:** `/api/attendance/*`, browser Geolocation API.

### Leaves (`/dashboard/leaves`)
- **Purpose:** leave requests, balances and calendar.
- **Components:** request form (dates, nature, half-day), balance cards, leave calendar, holiday list, medical-report upload, approval controls (for managers/admin).
- **Actions:** submit/delete leave, upload medical cert, approve/reject, manage holidays.
- **Dependencies:** `/api/leaves/*`.

### Chat (`/dashboard/chat`)
- **Purpose:** team messaging (1,107 LOC).
- **Components:** channel/DM sidebar, message list, composer with file upload, thread panel, @mention picker, unread badges.
- **Actions:** send/receive messages, open threads, mention users, create channels/DMs, attach files.
- **Dependencies:** `/api/chat/*`, Socket.IO (`newMessage`, mentions, threads).

### Expenses (`/dashboard/expenses`)
- **Purpose:** expense claims & approval (1,344 LOC).
- **Components:** claim form (category, structured fields, self-travel, bill upload), expense table, status badges, approval/pay controls, bulk actions, export.
- **Actions:** create claim, upload bill (→ Google Drive), approve/reject/pay, bulk approve/pay, export CSV.
- **Dependencies:** `/api/expenses/*`, `/api/expense-types`, driveService (server-side).

### TrainerLog (`/dashboard/trainer-log`)
- **Purpose:** trainer activity logging (trainer-only).
- **Components:** log form (org, type, mode, times), my-logs table.
- **Actions:** create log, view logs.
- **Dependencies:** `/api/trainer-log/*` (isTrainer-gated).

## Screens present but NOT routed

Two page files are **not registered** in `App.jsx`'s route table:

- **`AdminAttendance.jsx`** (690 LOC) — admin attendance oversight & correction. Its functionality (all-attendance view, corrections) exists via API but has no route entry, so it is either reached another way or currently orphaned.
- **`Timesheets.jsx`** (559 LOC) — worklog entry/review. Also has no route in `App.jsx`.

This is worth confirming during the upgrade: either wire these routes intentionally or treat them as dead UI (see [11 — Technical Debt](11-technical-debt.md)).

## Shared UI inventory

- **Layout:** `DashboardLayout` (sidebar, logout, toasts, heartbeat).
- **Widget:** `TeamStatusPanel` (live presence).
- **Marketing set:** Navbar, Hero, Features, HowItWorks, SocialProof, ChatHighlight, Testimonials, Pricing, CtaBanner, Footer.
- **Hook:** `useFadeIn`.

There is **no shared library of primitive components** (Button, Card, Table, Modal, FormField). These patterns are re-implemented inline within each large page — the single biggest UI-consistency and maintainability gap (see [12 — Reusable Components discussion in Performance/Tech Debt reports]).
